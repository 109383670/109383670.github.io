更新步骤

  1. 下载开发包（github也可以)
  2. 运行build目录下，cocos2d_tests.xcodeproj工程。
  3. 将编译Target选择成cpp-tests iOS
  4. 运行
  5. 用命令行命令生成一个模板工程

CCObject改成Ref

CCPoint改成Vec2

CCJumpTO等动作改成JumpTo

CCPointZero改成Vec2::Zero

除了0向量外，还有很多的静态值。

动作回调CallFuncN有改动

CC_CALLBACK_1 CC_CALLBACK_2 CC_CALLBACK_3
后面的数字表示带参数的多少。

ssize_t格式化参数%zd

ssize_t：有符号整数，与平台无关。适配不同平台的通用整数关键字，在32位平台为32，64位平台为long。
size_t：无符号整数，与平台无关。
ssize_t格式化参数是%zd, size_t是%tu。

Label Font

TTF, 系统字体，BMFont，都由Label类负责创建

错误记录

关掉 bitcode 在 buildsetting

NSAppTransportSecurity

注意类型NSAppTransportSecurity为Dictionary，NSAllowsArbitraryLoads为Boolean，复制粘贴的时候，不要多了空格，segment fault 页面上直接复制，经常会多一个出空格！值设为YES

require fullscreen 打勾，不然 iPad 会有分屏报错

https://itunes.apple.com/cn/developer/shuai-li/id944176950

Enable Modules 设置为 true，不然会有@import 的错误

非arc，在编译资源后加入 -fobjc-arc

Multiple commands produce

在 target -> Build phase > Copy Bundle Resource 中找到info.plist，移除

在 head 搜索路径中加入 googleadmob 的 framework 路径

Command CodeSign failed with a nonzero exit code

resource fork, Finder information, or similar detritus not allowed
文件签名失败，所有的资源文件要有签名，带com.apple.FinderInfo这个属性的文件，都要将改属性去掉。一般都是.png,.jpg 等图片。

  1. 查看目录下所有文件的签名,结果保存在.txt 中：

    1. 打开.txt 文件，查找有com.apple.FinderInfo这个属性的文件
    2. 去掉属性

也可以批量处理当前目录，一条一条执行：

iTunesArtwork这中没后缀的要单独处理。

GADInvalidInitializationException

libc++abi.dylib: terminating with uncaught exception of type NSException.
googleadmob 的 Id 问题，在 info.plist 中加入

新增加分辨率，需要制作对应的启动图片，不然新分辨率不起作用

参考：ipad,iphone启动图片命名 - ugg的专栏 - 博客频道 - CSDN.NET

NSLocationWhenInUseUsageDescription 用途声明

NSLocationWhenInUseUsageDescription
NSBluetoothPeripheralUsageDescription
同时声明
之前的蓝牙声明NSBluetoothPeripheralUsageDescription不要删除，不然 ios13 以下用不了
